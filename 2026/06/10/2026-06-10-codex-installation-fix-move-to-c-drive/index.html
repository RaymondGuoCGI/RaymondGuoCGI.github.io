避坑指南：Codex 安装后找不到 Computer Use 或 Chrome 插件？一招搞定

避坑指南：Codex 安装后找不到 Computer Use 或 Chrome 插件？

最近很多朋友在安装好 Codex 之后，兴奋地准备体验 Computer Use 功能或者安装 Chrome 插件，结果却发现：找！不！到！

明明安装过程一切正常，但在设置界面或者插件列表中就是刷不出来。或者在尝试手动安装插件时，系统反复报错，无法正常载入。

经过多方排查，我发现了一个非常“玄学”但却极其有效的解决方案。

1. 现象描述

  • 在插件市场无法找到 computer use 插件。
  • 在设置/电脑操控中显示无法安装插件。
  • 通常发生在你为了节省系统盘空间，将 Codex 安装到了 D 盘、E 盘或其他非系统盘的情况下。

2. 核心原因

Codex 的某些核心功能（尤其是涉及底层系统权限的 Computer Use 和浏览器扩展通信）对路径权限非常敏感。当应用安装在非系统盘时，Windows 的应用沙盒权限可能无法正确映射到相关组件，导致功能失效。

3. 终极解决方案：一键迁移至 C 盘

不需要卸载重装！Windows 自带的应用移动功能就能解决这个问题。

操作步骤：

  1. 按下键盘上的 Win + I 键，打开 Windows 设置。
  2. 点击左侧导航栏中的 应用 (Apps)。
  3. 选择 安装的应用 (Installed Apps)。
  4. 在应用列表中找到 Codex。
  5. 点击 Codex 右侧的 “三个小点” (…) 按钮。
  6. 选择 移动 (Move)。
  7. 在弹出的对话框中，选择 (C:) 盘，然后点击 移动。

等待系统进度条走完，Codex 就会被完整地迁移到系统盘。

4. 验证效果

迁移完成后，请彻底退出 Codex 并重新启动：

  1. 再次检查设置界面，你应该能看到 Computer Use 功能已经悄然出现。
  2. 再次尝试安装 Chrome 插件，此时安装程序应该能正确识别路径并顺畅完成。

总结

虽然把应用装在非系统盘是我们的好习惯，但对于 Codex 这种与系统底层高度集成的“怪兽”级工具，老老实实装在 C 盘才是最省心的做法。

如果你也遇到了类似的问题，赶紧试试这个方法吧！
