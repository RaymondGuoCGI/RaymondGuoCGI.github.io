效率翻倍：如何在 Codex 中开启多 Agent 协作模式

效率翻倍：如何在 Codex 中开启多 Agent 协作模式

在 AI 辅助开发的领域，单个智能体的能力终究有上限。Codex 最近推出的多 Agent 协作模式 (Multi-Agent Collaboration) 允许我们同时调用多个子智能体（Sub-agents）并行处理任务，比如让一个 Agent 写代码，另一个同步进行安全审查，第三个负责编写测试。

如果你发现你的 Codex 还是“单打独斗”，按照下面的步骤即可开启“群殴”模式。

1. 开启实验性功能

目前多 Agent 协作在某些版本中仍属于实验性功能，需要手动开启：

  1. 打开 Codex 桌面应用。
  2. 点击左下角的 Settings (设置) 图标。
  3. 在左侧菜单中找到 Features (功能) 或 Experimental (实验性) 选项卡。
  4. 找到 “Multi-agents” 或 “Sub-agents” 开关，将其勾选为开启状态。

提示：开启后可能需要重启应用以生效。

2. 通过指令触发协作

开启开关后，Codex 并不会在每个简单任务中都自动开启多 Agent。你需要通过特定的指令来“激活”这个能力：

  • 明确委派： 在 Prompt 中加入类似“请启动 3 个 Agent 分别处理…”的指令。
    • 示例： “帮我重构这个模块。请启动子 Agent 分别负责：1. 提取抽象类；2. 优化异常处理；3. 编写单元测试。”
  • 任务拆解： 当你提交一个极其复杂的任务时，Codex 现在会主动询问你：“是否需要拆解任务并分发给多个 Agent 协同处理？”点击确认即可。

3. 进阶配置：自定义团队规则

如果你想让多 Agent 模式更加智能化，可以在项目根目录配置一个 AGENTS.md 文件。这相当于给你的 AI 团队制定“员工手册”：

Codex 会自动读取这个文件，并在后续的协作中严格执行。

4. 并行安全的保障：Isolated Worktrees

在多 Agent 模式下，最担心的就是多个 AI 同时改同一个文件导致冲突。Codex 提供了一个关键功能：Isolated Worktrees (隔离工作树)。

在启动协作任务时，建议在任务设置中勾选此选项。Codex 会为每个子 Agent 创建一个独立的临时目录进行操作，只有在所有任务通过校验后，才会统一合并回主目录。

总结

多 Agent 协作不仅仅是“人多力量大”，更是专业化分工的体现。通过开启这个模式，你可以从一个“码农”晋升为“AI 项目经理”，指挥一队 AI 高效交付代码。

赶紧去你的设置里看看这个开关开了没有吧！
