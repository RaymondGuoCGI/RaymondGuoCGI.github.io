效率翻倍：如何在 Codex 中开启多 Agent 协作模式

效率翻倍：如何在 Codex 中开启多 Agent 协作模式

在 AI 辅助开发的领域，单个智能体的能力终究有上限。Codex 最近推出的多 Agent 协作模式 (Multi-Agent Collaboration) 允许我们同时调用多个子智能体（Sub-agents）并行处理任务。

如果你在设置中找不到对应的开关，请按照以下真实的操作界面步骤进行操作。

1. 使用“搜索设置”快速定位 (最推荐)

根据 Codex 最新的 UI 布局，设置项非常多，手动查找效率较低。

  1. 点击 Codex 左下角的 设置 图标。
  2. 在左侧边栏最上方的 “搜索设置…” (Search settings…) 输入框中，输入关键字：agent 或 智能体。
  3. 系统会自动过滤出包含“多智能体协作”或“子智能体”的选项，直接点击开启即可。

2. 手动查找路径

如果搜索没有结果，请检查以下两个位置：

  • 路径 A：常规 (General)
    点击左侧边栏顶部的 常规，在右侧页面下拉，查看是否有“开启实验性功能”或“Beta 功能”的勾选框。
  • 路径 B：配置 (Config)
    点击左侧边栏的 配置（带点的齿轮图标），这里通常存放着模型调用和高级编排的相关开关。查看是否有关于 Orchestrator 或 Multi-agent 的配置项。

3. 在聊天界面直接激活

很多时候，多 Agent 模式不需要在全局设置中开启，而是随用随开：

  • 使用 @ 召唤： 在聊天输入框中输入 @ 符号，你会看到一个智能体列表（如 @explore、@scout 等）。选择它们，Codex 就会以协作模式启动子任务。
  • 使用 / 指令： 输入 /collab 或 /agent，可以查看当前是否可以开启协作线程。
  • Composer 模式： 在聊天区域顶部或底部寻找 “Composer” 切换开关。进入 Composer 模式后，Codex 默认会以多智能体协作的方式来处理复杂的代码重构。

4. 关键点提示

  • 模型选择： 多 Agent 模式通常需要高性能模型支持。请确保你在对话中选择了 Claude 3.5 Sonnet 或 GPT-4o。
  • 工作树 (Worktrees)： 如果你开启了多 Agent，建议在侧边栏的 工作树 选项中查看并行任务的进度，这样可以防止不同 Agent 之间的代码修改发生冲突。
